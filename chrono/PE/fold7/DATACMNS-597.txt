BasicPersistentEntity.getPropertyAccessor for target bean does not properly handle inheritance when PersistentEntity instances are interface or abstract class based
The BasicPersistentEntity&amp;apos;s getPropertyAccessor(..) method fails to take into account &amp;apos;target&amp;apos; beans that implement the specified type of the PersistentEntity.  The specified type may be an interface or a abstract base class of the implementing domain object entity.
Spring Data GemFire, in particular, added support for interface-based PersistentEntity definitions in SGF-260, where the entity meta-data (such as @Id comes from an interface definition of the entity.
