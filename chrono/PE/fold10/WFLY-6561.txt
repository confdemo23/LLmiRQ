EJB Timers Intermittently Execute Repeatedly on Server Restart with Error Code WFLYEJB0043
On Wildfly10, we are experiencing intermittent problems where on server startup, an @Timeout annotated method can be executed hundreds of times in a second (log below).  Going through the references to the @Timeout methods and the @Scheduled methods, all scheduled methods are set to be non-persistent (ex: timerConfig.setPersistent(false), persistent=false) so we don&amp;apos;t know how this could occur.  It is not consistent and our only solution has been to kill the Wildfly10 server process and restart.
