Query derivation doesn't create joins for in-clauses on element collections
We currently don't support finderQueries for ElementCollection since we don't generate the necessary joins.
E.g.: The following use case dosen't work at the moment:
...
@ElementCollection private Set&lt;String&gt; attributes;
...
...
List&lt;User&gt; findByAttributesIn(Set&lt;String&gt; attributes);
...
