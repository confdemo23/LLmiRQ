JavaParserMutableClassOrInterfaceTypeDetails.createType(..) fails to create types with correct "implements" references
The JavaParserMutableClassOrInterfaceTypeDetails.createType(..) can be used to create new types. The arguments passed to the method include the list of types the new type should extend, and the list of types the new type should implement. Line 406 of JavaParserMutableClassOrInterfaceTypeDetails appears to be incorrectly handling the types to be implemented.
