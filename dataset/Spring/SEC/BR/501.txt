convertUrlToLowercaseBeforeComparison in PathBasedFilterInvocationDefinitionMap  is not consistent
if convertUrlToLowercaseBeforeComparison is true the url is converted to lower case before URL comparison but the pathMatcher still use the URL defined in the configurations without using toLowerCase and this may lead insorrect result
