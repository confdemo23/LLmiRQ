camel-linkedin - update_key option should be optional in getHistoricalStatusUpdateStatistics
update_key option should be optional in getHistoricalStatusUpdateStatistics endpoint. Currently it ends with an exception if the option is not specified: org.apache.camel.RuntimeCamelException: Missing properties for getHistoricalStatusUpdateStatistics, need one or more from [end_timestamp, update_key] See the LinkedIn API docs: https://developer-programs.linkedin.com/historical-company-statistics.
